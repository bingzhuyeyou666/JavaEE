# 陌路寻阡智慧文旅综合服务平台 V1.0 使用说明书

## 1. 软件概述

陌路寻阡是一套浏览器端智慧文旅服务系统。系统将景点查询、景点详情、路线规划、景点预约热线查询、旅行广场、足迹打卡、景点申报和后台审核集中在同一套应用中。系统不提供在线预约、预约签到或核销服务。

本说明书依据当前项目实际界面和接口编写。未配置第三方密钥时，地图、天气和智能问答相关功能会使用关闭状态或本地备用结果，不影响景点浏览、路线规划、预约热线查询、社区和后台管理等基础功能。

## 2. 运行环境

- 操作系统：Windows 10 或 Windows 11
- Java：JDK 8 及以上
- Maven：3.6 及以上
- Node.js：18 及以上
- 浏览器：Chrome、Edge 等现代浏览器
- 数据库：H2 文件数据库

## 3. 安装与启动

1. 将项目完整复制到本地目录。
2. 确认命令行可以执行 `java -version`、`mvn -version` 和 `npm -version`。
3. 双击项目根目录中的“启动陌路寻阡.bat”，或在项目根目录执行 `.\run.bat`。
4. 脚本完成前端构建和后端打包后，会启动服务。
5. 在浏览器访问 `http://localhost:8080/app/`。

如需启用百度地图或天气服务，应在启动前设置自己的环境变量：

```powershell
$env:BAIDU_MAP_ENABLED="true"
$env:BAIDU_MAP_API_KEY="自行申请的密钥"
$env:BAIDU_WEATHER_API_KEY="自行申请的密钥"
```

## 4. 用户登录

1. 点击页面右上角“游客模式”进入登录页。
2. 输入演示账号 `demo` 和密码 `demo123`。
3. 点击登录按钮。
4. 登录成功后，右上角显示当前用户名，并可使用打卡、发布和申报等用户功能；预约热线可直接在景点详情页查看。

## 5. 景点导览

1. 在顶部导航中点击“景点导览”。
2. 在搜索框输入景点名称或地址关键字。
3. 按页面提供的分类、评分等条件筛选景点。
4. 点击景点卡片中的“查看详情”进入详情页。
5. 详情页展示景点简介、地址、开放信息、票价、评分及相关服务入口。
6. 点击“加入路线”可将该景点加入路线规划清单。

## 6. 路线规划

1. 从景点卡片加入 2 至 5 个景点。
2. 点击顶部导航“路线规划”。
3. 核对已选景点，需要时删除或重新选择。
4. 点击生成路线按钮。
5. 系统按照景点坐标计算游览顺序，并显示各段距离。

路线结果用于行程参考，不替代道路交通部门或地图导航软件的实时导航。

## 7. 预约热线查询

1. 进入景点详情页。
2. 在“预约热线”区域查看该景点的官方联系电话。
3. 如需预约，用户自行拨打热线向景点咨询开放时段、票务和预约要求。
4. 本系统不创建预约记录，不生成核销码，也不提供预约签到或取消预约功能。

## 8. 足迹打卡

1. 登录后进入支持打卡的景点详情页。
2. 浏览器允许定位时，系统读取当前位置并计算与景点的距离。
3. 满足打卡距离条件后提交打卡。
4. 打卡成功后，可在个人中心查看足迹和已获得的勋章。

定位被拒绝或定位距离不满足要求时，系统会给出提示，不会生成有效打卡记录。

## 9. 旅行广场

1. 点击顶部导航“旅行广场”。
2. 浏览平台已有的旅行动态。
3. 登录后点击发布入口，填写标题、正文和分类，可按页面提示上传图片。
4. 在动态详情页查看评论，或提交自己的评论。
5. 点赞操作会更新对应动态或评论的点赞数量。

## 10. 景点申报

1. 登录后点击“景点申报”。
2. 填写景点名称、地址、类型、简介和申报理由。
3. 按页面提示上传相关图片。
4. 提交后，可在个人中心查看申报状态。
5. 管理员审核通过后，申报内容进入景点数据。

## 11. 管理后台

1. 访问 `http://localhost:8080/app/admin`。
2. 使用管理员账号 `admin` 和密码 `admin123` 登录。
3. 后台可维护景点信息、首页轮播和精选景点。
4. 在申报审核中查看用户提交的景点资料。
5. 审核通过会创建或更新对应景点；审核拒绝时应填写拒绝原因。

正式部署时必须通过环境变量修改默认管理员密码：

```powershell
$env:APP_ADMIN_USERNAME="自定义管理员账号"
$env:APP_ADMIN_PASSWORD="高强度密码"
```

## 12. 数据保存与退出

系统业务数据默认保存在项目根目录 `data` 文件夹的 H2 数据库文件中。关闭程序后数据仍会保留。删除该目录会使本地数据丢失，并在下次启动时重新初始化。

用户可通过页面中的退出按钮结束当前登录状态。关闭服务时，可关闭启动脚本对应的终端窗口。

## 13. 常见问题

### 13.1 页面可以打开，但天气或地图不可用

检查百度服务环境变量是否已配置，并确认第三方账号配额和网络连接正常。未配置时属于正常的离线运行状态。

### 13.2 修改前端代码后页面没有变化

进入 `frontend` 目录执行 `npm run build`，再重新启动后端服务。

### 13.3 端口被占用

确认 8080 端口没有被其他程序占用，或在 `application.yml` 中修改 `server.port` 后重新启动。

### 13.4 无法打卡

确认已经登录、浏览器已授予定位权限，并且当前位置满足景点打卡距离要求。
